Failure Playbook – quick triage Use when: JSON or images fail.

When something breaks, start here instead of poking randomly at HTML/JSON.

  • validate_env.py fails (missing images / bad paths)
    → Check the ENH_UPD / READY CSV Image_sub_url against the actual files in outputs/images/<SOP_ID> in SOP_Build. Fix names or paths, re-run validate_env.
  • Local 8080 test: JSON load error
    → In EdxBuild, grep the player for story.json and make sure the path is /EdxBuild/story/<SOP_ID>/story.json. If you see any /workspaces/ paths, run the sed patch step again and re-test.
  • JSON loads but no images in browser
    → Confirm GitHub Pages can see the image URL you are using, e.g. https://USERNAME.github.io/EdxBuild/site/BUILD/SOP/images/SE/Distro/LineEnt2/S000.png. If that works, the player’s image path needs adjusting; update Image_sub_url in ENH_UPD/READY, rebuild story and player.
  • Everything works locally, fails on GitHub Pages
    → Check that you committed the correct files in EdxBuild (story, player, images, index). Run git status to confirm only expected files are staged before push.
